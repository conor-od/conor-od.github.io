    Zola | Quick Start Guide

    Creating a static website using Zola.

    Prerequisites

    • GIT (Version Control)

    Step 1: Install Zola

    First we need to install Zola. There are several ways to do this which we can read about here in the official Zola docs. To keep it simple, below are the easiest and most popular ways to install Zola.

    Linux

    With Snap:

    snap install --edge zola

    Windows

    With Chocolatey:

    choco install zola

    Mac OS

    brew install zola

    Step 2: Initialise Project

    Next we need to create a new project. To do this we will initialise a new GIT repository and Zola project.

    Let's first create a new directory for our project. We will refer to this directory as the project_root from here on. Once we have created the project_root directory, we can open a terminal, and run the following commands:

    • Change directory to project_root

      cd <project_root>

    • Initialise GIT repository

      git init

    • Initialise Zola project

      zola init

      When prompted, set the URL to http://localhost:8080 and answer n to the follow up configuration prompts.

    We should now see the following directory structure:

    <project_root>
├── content
├── sass
├── static
├── templates
├── themes
└── config.toml

    Step 3: Test our project

    Before we go any further, lets run a quick test by building and deploying the project locally, and viewing Zola's default landing page.

    To do this we run the following commands:

    • Build the project

      zola build

    • Deploy the project locally

      zola serve --base-url localhost --port 8080

    We can now open our web browser and browse to localhost:8080. If our project built and deployed correctly, we should see the following landing page:

    Step 3: Setting up a Theme

    Now we have our project setup, we can begin the process of building our website. First we will choose a theme, then we will add it to our project and enable it. For the purposes of this tutorial we will be using the blow theme. For reference, more themes can be found here.

    Get a Theme

    We will add the blow theme to our project as a git submodule as opposed to checking it out as a repository inside our project repository, as the official docs suggest. To do this we can run the following commands:

    • Change directory to <project_root>/themes
      cd <project_root>/themes
    • Add blow theme as a git submodule
      git submodule add -b main git@github.com:tchartron/blow.git blow

    We should now have the following directory structure:

    <project_root>
├── content
├── sass
├── static
├── templates
├── themes
    └── blow
└── config.toml

    Enable Theme

    To enable a theme, we set the theme property at the top level of the config.toml to match the folder name of the theme.

    To enable our blow theme we need to add the following property to our config.toml:

    # The site theme to use.
theme = "blow"

    Now we can refresh the webpage (localhost:8080) and observe it has changed to the blow theme, like so:

    Step 4: Configure Theme

    Next we can modify our themes content by changing configuration in our config.toml. Below is example configuration for a simple one page website we can use add to our config.toml:

    [extra]
enable_search = false
enable_sidebar = false
enable_multilingue = false
enable_adsense = false

[extra.navbar]
title = "J-Doe"
items = []

# Index page
[extra.index]
title = "John Doe"
image = "https://conor-od.github.io/blog/zola-quick-start/default-logo.png"
image_alt = "Website Logo"

[extra.social]
github = "https://github.com/johndoe"
linkedin = "https://www.linkedin.com/in/johndoe/"
email = "john.doe@email.me"

    Now we can refresh the webpage (localhost:8080) and observe our configuration has been applied, like so:

    Step 5: Extending a Theme

    Finally, we will make a small change to our index page beyond what our theme supports out of the box. This change will be adding another social media link icon to the homepage alongside the Github, Linkedin and Email icons.

    To achieve this first we need to override the theme file we want to change with our own version. To do this we add a file of the same name to our <project_root>/templates directory, in this case we will be overriding the index.html file. As we only want to modify the blow theme's index.html, lets copy the theme's version of the file, and then modify it. To do this copy the blow theme's <project_root>/themes/blow/templates/index.html to our projects template directory <project_root>/templates/index.html.

    Once copied, we should have the following directory structure:

    <project_root>
├── content
├── sass
├── static
├── templates
    └── index.html
├── themes
    └── blow
└── config.toml

    Now we can add our stackoverflow icon change by adding a snippet of html to our new index.html file, and a configuration property to our config.yaml as shown below:

    • Add the following to <project_root>/templates/index.html:

          {% if config.extra.social.stackoverflow %}
    <a href="{{config.extra.social.stackoverflow}}" target="_blank">
        <svg xmlns="http://www.w3.org/2000/svg" class="h-12 w-12 text-black dark:text-white" viewBox="0 0 18 18" fill="currentColor" stroke="none">
        <path d="M12.412 14.572V10.29h1.428V16H1v-5.71h1.428v4.282h9.984z"/> <path d="M3.857 13.145h7.137v-1.428H3.857v1.428zM10.254 0 9.108.852l4.26 5.727 1.146-.852L10.254 0zm-3.54 3.377 5.484 4.567.913-1.097L7.627 2.28l-.914 1.097zM4.922 6.55l6.47 3.013.603-1.294-6.47-3.013-.603 1.294zm-.925 3.344 6.985 1.469.294-1.398-6.985-1.468-.294 1.397z"/>
        </svg>
    </a>
    {% endif %}

    • Add a link to our stackoverflow profile in the config.toml configuration, along side our other social media links from step 4:

      [extra.social]
stackoverflow = "https://stackoverflow.com/users/1234/john-doe"

    Now we can refresh the webpage (localhost:8080) and observe our new stackoverflow icon, like so:

    Conclusion

    That's it. We have built our first static site using Zola!

    Zola is extremely easy and fast way to get a simple static site up and running. I will be using it for building conorod.me and include more posts on how I do that here.
